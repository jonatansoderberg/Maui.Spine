## The three-file pattern

Every page consists of exactly **three files** that are kept together as a logical node in the project file using `<DependentUpon>`:

```
Pages/
  MyPage.cs                ← code-behind  (auto-included by MSBuild glob)
  MyPage.View.xaml         ← XAML layout  (must be listed in .csproj)
  MyPage.ViewModel.cs      ← ViewModel    (must be listed in .csproj)
```

> **Why the explicit entries?**
> The project file contains `<Compile Remove="Pages\**\*.ViewModel.cs" />`, which excludes **all** ViewModel files from compilation by default. Each ViewModel — and each XAML file — must therefore be opted back in with explicit `<Compile Include>` / `<MauiXaml Update>` entries and a `<DependentUpon>` tag so that all three files nest under the same code-behind node in Solution Explorer.

---

## 1. Code-behind — `MyPage.cs`

A minimal partial class decorated with either `[NavigableRegion]` or `[NavigableSheet]`.

```csharp
namespace App.Pages;

[NavigableRegion(Title = "My page")]
public partial class MyPage { public MyPage() => InitializeComponent(); }
```

This file is the **anchor** for the other two — its filename (without extension) is the value used in `<DependentUpon>` inside the project file.

---

## 2. XAML layout — `MyPage.View.xaml`

The root element must always be `<SpinePage>`. Bind the ViewModel through `x:TypeArguments` and `x:DataType`.

```xml
<SpinePage
    xmlns="http://schemas.microsoft.com/dotnet/maui/global"
    xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml"
    x:Class="App.Pages.MyPage"
    x:TypeArguments="MyPageViewModel"
    x:DataType="MyPageViewModel">

    <!-- layout content -->

</SpinePage>
```

The global MAUI xmlns (`http://schemas.microsoft.com/dotnet/maui/global`) already covers the following namespaces via `GlobalXmlns.cs`, so no extra `xmlns:` declarations are needed for them:

| Namespace | Source |
|---|---|
| `Plugin.Maui.Spine.Core` | `Plugin.Maui.Spine` assembly |
| `Plugin.Maui.Spine.Presentation` | `Plugin.Maui.Spine` assembly |
| `Plugin.Maui.SvgImage` | `Plugin.Maui.SvgImage` assembly |
| `App.Pages` | this project |
| `App.Pages.Settings` | this project |

## 3. ViewModel — `MyPage.ViewModel.cs`

Inherits `ViewModelBase` (from `Plugin.Maui.Spine.Core`). Use primary-constructor syntax for service injection. Decorate observable state with `[ObservableProperty]` and commands with `[RelayCommand]` — both are source-generated by `CommunityToolkit.Mvvm`.

```csharp
namespace App.Pages;

public partial class MyPageViewModel(INavigationService _navigation) : ViewModelBase
{
    [ObservableProperty]
    private string? _someValue;

    [RelayCommand]
    private async Task DoSomething() => await _navigation.NavigateToAsync<OtherPage>();
}
```

The following namespaces are available everywhere via `GlobalUsings.cs` without any `using` statement:

```csharp
global using CommunityToolkit.Mvvm.ComponentModel;  // [ObservableProperty]
global using CommunityToolkit.Mvvm.Input;            // [RelayCommand]
global using Plugin.Maui.Spine.Core;                 // INavigationService, ViewModelBase, etc.
```

## 4. Register in the project file

Open `MauiBottomSheetPoc.csproj` and add the following two entries inside the `<ItemGroup>` that contains all the other page registrations:

```xml
<MauiXaml Update="Pages\MyPage.View.xaml">
  <Generator>MSBuild:Compile</Generator>
  <DependentUpon>MyPage.cs</DependentUpon>
</MauiXaml>

<Compile Include="Pages\MyPage.ViewModel.cs">
  <DependentUpon>MyPage.cs</DependentUpon>
</Compile>
```

The code-behind file (`MyPage.cs`) does **not** need an entry — it is picked up by the default MSBuild glob.

## Page types

### Region page

A region page is pushed onto the navigation stack and displays with a header bar (title, back button, action buttons).

```csharp
[NavigableRegion(Title = "My page")]
public partial class MyPage { public MyPage() => InitializeComponent(); }
```

Common optional properties:

| Property | Type | Default | Description |
|---|---|---|---|
| `Title` | `string` | `""` | Header bar title |
| `Lifetime` | `ServiceLifetime` | `Transient` | DI lifetime |
| `IsHeaderBarVisible` | `bool` | from `SpineOptions` | Show/hide header bar |
| `IsBackButtonVisible` | `bool` | from `SpineOptions` | Show/hide back button |
| `TitleAlignment` | `TitleAlignment` | from `SpineOptions` | `Left` / `Center` |
| `TitlePlacement` | `TitlePlacement` | from `SpineOptions` | `Inline` / `Large` |

### Bottom-sheet page

A bottom-sheet page slides up over the current content.

```csharp
[NavigableSheet(
    Title = "My sheet",
    BackgroundPageOverlay = BackgroundPageOverlay.Dimmed,
    AllowedDetents = [SheetDetent.Medium, SheetDetent.FullScreen])]
public partial class MyPage { public MyPage() => InitializeComponent(); }
```

Additional properties beyond those shared with `[NavigableRegion]`:

| Property | Type | Description |
|---|---|---|
| `BackgroundPageOverlay` | `BackgroundPageOverlay` | `None`, `Dimmed`, or `Blurred` |
| `AllowedDetents` | `object[]` | Array of `SheetDetent` values or percentage strings (e.g. `"75%"`) |

## Page lifecycle hooks

Override these methods in the ViewModel to react to navigation events:

```csharp
// Called every time the page appears (forward navigation and return from stack)
public override Task OnAppearingAsync(NavigationDirection navigationDirection)
{
    return base.OnAppearingAsync(navigationDirection);
}

// Called when the user attempts to close/dismiss the sheet
// Return false to block dismissal
public override Task<bool> OnCloseRequestedAsync() => Task.FromResult(true);
```

`NavigationDirection` values: `Forward`, `Back`.

---

## Header bar action buttons

Add `PageAction` items inside `OnAppearingAsync`. Guard with `PageActions.Count == 0` to prevent duplicates when re-navigating back to a page.

```csharp
public override Task OnAppearingAsync(NavigationDirection navigationDirection)
{
    if (PageActions.Count == 0)
    {
        // text-only button
        PageActions.Add(new PageAction(text: "Save", command: SaveCommand));

        // icon-only button (SVG from Resources/Images)
        PageActions.Add(new PageAction(text: null, command: OpenSettingsCommand) { Svg = "settings.svg" });
    }
    return base.OnAppearingAsync(navigationDirection);
}
```

## Navigation

`INavigationService` is injected via the ViewModel constructor. No route registration is needed — Spine auto-discovers all `[NavigableRegion]` and `[NavigableSheet]` pages from the assembly passed to `options.AddAssembly(...)` in `MauiProgram.cs`.

```csharp
// Push a region page
await _navigation.NavigateToAsync<SomePage>();

// Open a bottom sheet
await _navigation.NavigateToAsync<SomeSheetPage>();
```

## Subdirectory pages

Pages in a subfolder follow the same three-file pattern but use a matching namespace and subfolder paths in the project file.

**Namespace:** `App.Pages.<FolderName>` (e.g. `App.Pages.Settings`)

**Project file entries:**

```xml
<MauiXaml Update="Pages\Settings\SettingsPage.View.xaml">
  <Generator>MSBuild:Compile</Generator>
  <DependentUpon>SettingsPage.cs</DependentUpon>
</MauiXaml>

<Compile Include="Pages\Settings\SettingsPage.ViewModel.cs">
  <DependentUpon>SettingsPage.cs</DependentUpon>
</Compile>
```

If the new namespace is not yet mapped in `GlobalXmlns.cs`, add an entry so it is available in XAML without an explicit `xmlns:` declaration:

```csharp
[assembly: XmlnsDefinition(
    "http://schemas.microsoft.com/dotnet/maui/global",
    "App.Pages.MyFolder")]
```

## Checklist for a new page

- [ ] Create `Pages\MyPage.cs` with `[NavigableRegion]` or `[NavigableSheet]` and `InitializeComponent()`
- [ ] Create `Pages\MyPage.View.xaml` with `<SpinePage>` root, correct `x:Class`, `x:TypeArguments`, and `x:DataType`
- [ ] Create `Pages\MyPage.ViewModel.cs` inheriting `ViewModelBase`
- [ ] Add `<MauiXaml Update>` entry in `MauiBottomSheetPoc.csproj` with `<DependentUpon>MyPage.cs</DependentUpon>`
- [ ] Add `<Compile Include>` entry in `MauiBottomSheetPoc.csproj` with `<DependentUpon>MyPage.cs</DependentUpon>`
- [ ] If the page is in a new subfolder, add the namespace to `GlobalXmlns.cs`
